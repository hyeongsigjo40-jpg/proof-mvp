# Proof MVP 빌드 프롬프트 (Codex용)

## 제품 한 줄 요약

Proof는 작심삼일의 정확한 트리거 지점을 찾아, 사전 실행계획·외부 확인·누적기록으로 묶어주고,
그 결과를 다시 메커니즘 진단에 되먹여 점점 더 정확해지는 습관실행 서비스다.

## 타겟 사용자 (포지셔닝)

본인이 게으른 완벅주의 성향이 있다고 말하며, 자기계발 개선을 위해 GPT 상담이나 계획수립
방법론(뽀모도로 등)을 적용해본 적 있는 20대. 장기 목표와 연결되는 공부 습관을 매일/주 단위로
만들려 시도하지만 3일 이상 지속하지 못하고 작심삼일로 끝난다.

차별화: GPT/뽀모도로(보편적 조언, 개인화 없음) / AI감정일기형 앱(진단 후 행동개입 루프 없음) /
예치금형 챌린지 앱(외부압박은 있으나 개인화·트리거설계 없음)와 달리, Proof는 첫 진단에 멈추지
않고 실제 개입의 성공/실패 데이터를 계속 모아 메커니즘 진단 자체를 갱신한다.

---

## 핵심 아키텍처: Layer 0 → 1 → 2

```
Layer 0 (최소 입력, 온보딩에서 1회)
    ↓
Layer 1 (핵심 루프 — 매일/매주 반복되는 실제 기능)
    ↓
Layer 2 (피드백 — 1의 결과가 다시 0의 진단을 갱신)
```

### Layer 0 — 최소 입력 (절대 깊게 가지 않을 것)

다음 정보만 받는다. 심리분석 리포트, 24장 카드, 무의식 해석 같은 것을 만들지 말 것
(경쟁사 마인드플래닛이 여기서 멈추는 함정 — Proof는 여기를 출발점으로만 쓴다):

- 만들고 싶은 습관 (자유 텍스트, 1개만 — 예: "주 3회 토익 공부")
- 보통 무너지는 시간대/상황 (선택형: 아침 기상 직후 / 저녁 퇴근·하교 후 / 마감 직전 / 기타-자유입력)
- 평소 무너질 때 본인이 하는 행동 (자유 텍스트, 1줄 — 예: "숏폼 보다가 시간 다 씀")

이 3개 입력값으로 충분하다. 더 캐묻지 않는다.

### Layer 1 — 핵심 루프 (매일 반복되는 기능, 이번 MVP의 본체)

세 가지 기능을 하나의 "저녁 회고" 흐름으로 묶는다:

**A1. 사전 if-then 계획 (저녁 회고 안에서)**
- 매일 저녁 기본 21:00 알림 → 체크인 후 "내일 [언제], [어디서], 정확히 [무엇을] 할지" 한 문장 입력
- 설정에서 시간 선택 UI로 회고 시간을 바꿀 수 있음
- 자유 텍스트 한 줄로 충분 (예: "저녁 8시, 책상에서, 토익 RC 10문제")
- 입력 안 하면 다음 저녁 회고에서 "확인할 계획이 없어요" 정도로만 가볍게 알려줄 것
  (죄책감 유발 카피 금지 — 아래 디자인 제약 참조)

**A9. 저녁 체크인**
- 매일 설정된 시간에 1회 알림: "계획한 거 하셨어요?" → Yes / No / 일부만
- No 또는 일부만 선택 시, 추가 질문 절대 캐묻지 말고 단 1개만:
  "그 순간 뭘 하고 있었어요?" (자유 텍스트, 선택 가능 — 건너뛰기 허용)
- 응답 없이 다음날로 넘어간 계획은 실패로 임의 판정하지 않고 no_response로 구분
- 평가성 질문 금지 (아래 디자인 제약 참조)

**A3. 트랙레코드**
- 캘린더/리스트 형태로 지금까지의 Yes/No/일부 기록을 누적해서 보여줌
- "이번 주 N번 중 M번 했음" 같은 단순 누적 수치만, 점수화·등급화·배지 없음
- partial은 완료 카운트에 포함하고, done / partial / no_response / not_done 네 상태를 구분 표시
- 무너진 날 다시 시작할 때 이 화면을 먼저 보여줘서 "이미 해온 것"을 환기

### Layer 2 — 피드백 (메커니즘 진단 갱신)

- A9에서 모인 "그 순간 뭘 하고 있었어요?" 답변들을 누적
- 일정 횟수(예: 5회) 쌓이면, 가장 자주 등장한 단어/상황을 사용자에게 보여줌:
  "최근에 무너진 순간들을 보면 [공통 패턴]이 자주 보여요"
- 이건 AI의 깊은 분석이 아니라 단순 빈도 집계로 시작해도 됨 (MVP는 통계만, NLP 고도화는 다음 버전)
- 이 패턴을 다음 A1(전날 밤 계획) 화면에 살짝 노출:
  "지난번엔 [상황]에서 자주 무너졌어요, 오늘은 그 시간대를 어떻게 다르게 보낼까요?"

## 디자인 제약 (반드시 지킬 것 — 위반시 핵심 가치제안이 깨짐)

1. **평가형 카피 금지**: "오늘 잘했어요/못했어요", "당신은 의지가 부족해요" 같은 정체성 평가
   문구를 절대 쓰지 않는다. 모든 카피는 "지금 무엇을 할까"에 초점 (자아개입 아닌 과제개입).
   예: "오늘 못했네요" (X) → "내일은 뭘 해볼까요?" (O)
2. **게이미피케이션 금지**: 포인트, 배지, 레벨, 스트릭 강조, 경쟁 랭킹 만들지 않는다
   (외적 보상의 과잉정당화 효과로 내재적 동기를 깎아먹을 위험).
3. **무거운 강제성 금지**: 금전 예치, 벌금, 그룹 동반책임 같은 기능은 이번 MVP에 넣지 않는다
   (반발심 유발 위험, 추후 검증 후 별도 검토).
4. **단계적 성공조건 (선택 구현)**: 여유 있으면 A1 계획에 "최소 버전"을 같이 입력받아도 좋음
   (예: "정상: RC 10문제 / 최소: RC 3문제만"). 시간 부족하면 다음 버전으로 미뤄도 됨.
5. **Layer 0를 절대 키우지 않는다**: 온보딩에 질문을 추가하고 싶은 유혹이 들면 멈출 것.
   진단은 Layer2(피드백)에서 점진적으로 쌓이는 것이 핵심 가치이지, 처음에 다 캐묻는 게 아님.

## 데이터 모델 (최소)

```
User
  - id
  - habit_name (string)
  - usual_breakdown_context (string)
  - usual_breakdown_behavior (string)
  - kakao_linked (bool, default false)
  - kakao_access_token (string, nullable)
  - kakao_refresh_token (string, nullable)
  - checkin_time (time, default '21:00')

DailyPlan
  - id, user_id, date
  - when_text, where_text, what_text (if-then 한 줄)
  - created_at

CheckIn
  - id, plan_id
  - result (enum: done / not_done / partial / no_response)
  - context_text (nullable, "그 순간 뭘 하고 있었어요" 답변)
  - created_at

PatternInsight (Layer2 산출물)
  - id, user_id
  - pattern_summary (string, context_text들의 빈도 집계 결과)
  - generated_at
```

---

## 화면 (MVP 최소 구성, 4개면 충분)

1. **온보딩** — Layer0 입력 3개 받기 (1페이지)
2. **저녁 회고** — 오늘 체크인 → PatternInsight → 내일 계획 입력
3. **트랙레코드** — A3 (캘린더 또는 리스트, 단순 누적 수치)
4. **설정** — 회고 시간 선택, 카카오 나에게 보내기 연결

---

## 기술 스택 제안 (MVP, 빠르게 만들 것이 목적)

- 웹앱 우선
- 알림은 카카오 "나에게 보내기" API 우선, 미연동 사용자는 웹푸시 폴백
- 프론트: 단순 React 또는 동급 경량 프레임워크
- 백엔드: 가벼운 API + SQLite/Postgres 중 택1
- AI/NLP는 MVP 단계에서 불필요 — PatternInsight는 단순 키워드 빈도 집계로 시작
- 인증은 이메일 매직링크 정도로 단순화 (소셜로그인 등 생략 가능)

---

## 명시적으로 만들지 말 것 (Scope 제외)

- 심리분석 리포트, 성격유형, 무의식 해석 같은 콘텐츠
- 포인트/배지/레벨/랭킹
- 금전 예치, 벌금, 그룹 챌린지
- 평가성 알림 문구 ("오늘 실패하셨네요" 류)
- 복잡한 온보딩 설문 (3개 입력 초과)
